# Indian News Comparator

A Chrome extension that analyses the political bias of any Indian news article you are reading, finds alternative perspectives from Left, Center, and Right sources, and explains the framing differences using a custom-trained ML model and an LLM.

---

## How It Works

1. You open a news article in Chrome and click **Get News** in the extension popup.
2. The extension extracts the article text and sends it to the backend.
3. **Port 8000 — Main API** (`api/main.py`): classifies the article's bias, extracts named entities, and fetches related articles from NewsAPI / The Guardian.
4. **Port 8001 — ML Engine** (`api/engine.py`): runs the article through the full ML pipeline to produce a bias score, retrieves a semantically similar alternative article from ChromaDB, and calls the Groq LLM to generate a plain-English explanation.
5. The extension displays the bias score on a −5 to +5 slider, Left / Center / Right perspective panels, and a cross-article comparison section.

---

## ML Architecture

The bias scoring pipeline has three stages:

```
Article text
    │
    ▼
IndicBERT  (ai4bharat/indic-bert)
    │  last_hidden_state  (batch × 512 × 768)
    ▼
LiquidBrain  (Liquid Neural Network — LTC with AutoNCP wiring, 64 neurons)
    │  final narrative state  (batch × 32)
    ▼
BiasClassifier  (Linear 32 → 3)
    │  softmax probabilities  [P_left, P_center, P_right]
    ▼
Bias Score  =  (P_right − P_left) × 5   ∈  [−5, +5]
```

- **IndicBERT** is an ALBERT-based transformer pre-trained on Indian languages and English. It captures the semantic meaning of Indian political vocabulary.
- **LiquidBrain** is a biologically-inspired Liquid Time-Constant (LTC) network. Unlike mean-pooling, it processes the full token sequence in order, capturing how framing evolves across the article.
- **BiasClassifier** is a single linear layer trained to map the LiquidBrain's output to Left / Center / Right probabilities.

### Training

The model was trained on 2,594 labeled Indian news article snippets using a two-stage approach:

| Stage | Epochs | What trains | Learning rate |
|---|---|---|---|
| 1 | 3 | LiquidBrain + classifier head only (IndicBERT frozen) | 1e-3 |
| 2 | 2 | End-to-end fine-tune (IndicBERT last layer + pooler unfrozen) | 2e-5 (BERT) / 1e-3 (head) |

Final accuracy: **65.4%** on a 3-class problem (random baseline = 33%).

## Setup

### Prerequisites

- Python 3.10 or higher
- Google Chrome
- API keys for:
  - [Groq](https://console.groq.com) — free tier available
  - [The Guardian Open Platform](https://open-platform.theguardian.com/access/) — free

### 1. Clone the repository

```bash
git clone https://github.com/shriyatallapragada/indian-news-comparator.git
cd indian-news-comparator
```

### 2. Create and activate a virtual environment

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
python3 -m spacy download en_core_web_sm
```

### 4. Set up environment variables

```bash
cp .env.example .env
```

Edit `.env` and fill in your keys:

```
GUARDIAN_API_KEY=your_guardian_api_key_here
GROQ_API_KEY=your_groq_api_key_here
```

### 5. Train the model

This generates the model weight files (`train/bias_classifier.pth` and `train/fine_tuned_bias_model.pth`). Takes approximately 15–20 minutes on Apple Silicon (MPS) or a modern CPU.

```bash
cd train
python3 train_full.py
cd ..
```

Expected output at the end:
```
── Stage 2 Epoch 2 complete | Loss: 0.79 | Acc: 65.4%
[BiasClassifier] Saved to .../train/bias_classifier.pth
Training complete.
```

### 6. Seed the vector database

```bash
cd api
python3 -c "from engine import initialize_database; initialize_database()"
cd ..
```

## Running the Extension

### Start both backend servers

```bash
./start.sh
```

This opens two Terminal tabs automatically:
- **Tab 1** — Main API on port 8000
- **Tab 2** — ML Engine on port 8001

Or start them manually:

```bash
# Terminal 1
cd api && uvicorn main:app --host 127.0.0.1 --port 8000 --reload

# Terminal 2
cd api && uvicorn engine:app --host 127.0.0.1 --port 8001 --reload
```

### Load the Chrome extension

1. Open Chrome and go to `chrome://extensions`
2. Enable **Developer mode** (toggle in the top right)
3. Click **Load unpacked**
4. Select the `extension/` folder from this repository
5. Pin the extension to your toolbar

### Use it

Navigate to any Indian news article, click the extension icon, and press **Get News**.

## API Reference

### Port 8000 — Main API

| Method | Endpoint | Description |
|---|---|---|
| POST | `/api/analyze` | Classify bias and extract entities from article text |
| POST | `/api/related` | Find related Left/Center/Right articles from ChromaDB |
| POST | `/api/ingest` | Ingest an article into ChromaDB |
| GET | `/news` | Fetch Left/Center/Right articles from NewsAPI/Guardian |

### Port 8001 — ML Engine

| Method | Endpoint | Description |
|---|---|---|
| POST | `/analyze_perspective` | Full ML pipeline — bias score, alternative article, LLM insights |
| POST | `/api/ingest_live_batch` | Batch ingest articles into the engine's ChromaDB |

## Key Design Decisions

**Why IndicBERT?** Standard English BERT models underperform on Indian political text because they lack exposure to Indian English idioms, party names, and regional political vocabulary. IndicBERT is pre-trained on 12 Indian languages plus English.

**Why a Liquid Neural Network?** Mean-pooling BERT's output treats all tokens equally. The LTC network processes tokens sequentially with continuous-time dynamics, meaning it can detect how framing shifts across a paragraph — a key signal in political bias.

**Why Groq + LLaMA for explanations?** The ML model produces a score but cannot explain it in plain English. The LLM receives the score and both article texts and generates a one-sentence explanation grounded in specific word choices, making the output interpretable.

**Why ChromaDB?** Semantic vector search allows the engine to find thematically related articles even when they share no exact keywords — important for matching Indian political stories across sources with different naming conventions.
